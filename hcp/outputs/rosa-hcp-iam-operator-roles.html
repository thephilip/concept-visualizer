ROSA HCP IAM Operator Roles & Route53 DNS
How the control-plane-operator assumes an IAM role via OIDC/IRSA to manage private hosted zone DNS records — and why CloudTrail alerts for this are expected.
⚠ ROSA HCP-specific. IAM operator roles and Route53 private hosted zone management via IRSA apply to ROSA with Hosted Control Planes only. ROSA Classic uses a different IAM model (OCM service roles) and does not create per-cluster operator roles. Self-managed OCP and ARO (which uses Azure Managed Identity) have separate identity architectures.

CloudTrail alert suppression
Alert sourceChangeResourceRecordSets on *.hypershift.local zones
Why it firesCPO reconciles DNS continuously — not on a fixed schedule
Suppress byAllowlist <cluster>-kube-system-control-plane-operator IAM role in CloudTrail rule
Pattern<PREFIX>-kube-system-control-plane-operator (one per cluster)
Is it a security issue?No — role is scoped to *.hypershift.local records only via ROSAControlPlaneOperatorPolicy
