ARO DNS Flow
How DNS queries travel pods to the Internet and possible failure points.
⚠ ARO-specific. The dnsmasq node-level forwarder and the 40-second timeout/blocking behavior described here apply only to Azure Red Hat OpenShift. On ROSA and self-managed OCP, CoreDNS forwards external queries directly to the VPC/cloud DNS resolver — no dnsmasq hop exists and this failure mode does not apply.
