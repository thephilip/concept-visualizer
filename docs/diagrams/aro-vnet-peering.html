Peering Properties
AllowVirtualNetworkAccessInclude peered VNet in virtual_network NSG tag
AllowForwardedTrafficAllow traffic forwarded from NVA through peering
AllowGatewayTransitHub side — let spokes use this VNet's gateway
UseRemoteGatewaysSpoke side — use hub's gateway instead of own
Key Constraints
Transitive routingNon-transitive — VNet A cannot reach VNet B via Hub without direct peering or NVA routing
Overlapping CIDRsNot allowed between peered VNets
One gateway per VNetUseRemoteGateways and local gateway are mutually exclusive
Max peers500 (1,000 with Azure VNET Manager)
ARO Considerations
Pod CIDRDefault 10.128.0.0/14 — must not overlap peered VNets
Service CIDRDefault 172.30.0.0/16 — check on-prem ranges
UDR + NVA0.0.0.0/0 via NVA routes all ARO egress through NVA — ensure required endpoints are allowed
NSG interactionARO NSG is immutable — UDR changes must not conflict
