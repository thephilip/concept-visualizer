🔭 Concept Visualizer — Help

What is this?

Concept Visualizer is a collection of interactive architecture reference diagrams for ARO, ROSA, HCP, and self-managed OCP — built for Support Engineers working with OpenShift, ARO, and ROSA.

Each diagram explains a specific flow, architecture, or failure mode with clickable components, detail cards, and quick-reference tables.

Navigating a diagram

Click any node in the flow diagram to expand its detail panel — covering role, key behavior, gotchas, and diagnostic commands.

Scope banner — the amber banner at the top of each diagram tells you which platform(s) the content applies to. Always read it before sharing with a customer.

Reference cards below the diagram cover timing, config commands, platform-specific facts, and diagnostics at a glance.

Verifying accuracy

Diagrams can go stale as products evolve. Run /visualize fact-check [file] in Claude Code to verify all factual claims and check every link against official sources.

Links are verified in parallel when there are more than 5 — it will flag anything outdated, dead-linked, or unverifiable and suggest corrections. Fact-check also runs automatically whenever a new diagram is built or compiled.

Theme

Use the ☾ Dark / ☀ Light toggle in the top-right corner. Your preference is saved across sessions.

Requirements

You need Claude Code with this repository cloned and open as the working directory.

Generate a new diagram

Run the /visualize slash command with a topic description:

/visualize ARO egress lockdown and gateway proxy architecture

Claude will read the spec, research official sources, confirm the platform scope with you, and build a complete one-pager with a flow diagram, detail cards, and reference tables.

💡 Tip: Attach a screenshot, whiteboard photo, or existing flowchart — Claude will use it as the structural basis for the diagram layout and expand it with accurate detail.

Platform routing

Diagrams are organized by platform. Claude will save your file to the correct folder automatically:

aro/ — ARO Classic  ·  hcp/ — ROSA HCP, ARO HCP, HyperShift  ·  rosa/ — ROSA Classic  ·  ocp/ — Self-managed OCP

Verify accuracy before publishing

Fact-check runs automatically after every new diagram is built — no manual step needed. It verifies all external links (in parallel when there are more than 5) and checks factual claims against official sources. Any dead links or incorrect claims must be resolved before the diagram is delivered.

You can also run it manually at any time: /visualize fact-check [file]

Publish to this site

Run /visualize compile — it fact-checks any new or changed files first, then inlines all assets and updates the docs/ directory. Push to GitHub to deploy.

Current workflow

Create your diagram using /visualize [topic], run /visualize compile, then submit a pull request to the repository.

If you don't have write access, open an issue on GitHub with your completed HTML file attached and a maintainer will review and publish it.

Accuracy requirements

All content must be traceable to an official source — docs.redhat.com, docs.openshift.com, access.redhat.com/solutions/, or learn.microsoft.com for ARO. The /visualize skill handles research automatically.

Every platform-specific diagram must include a scope banner. The skill adds one automatically.
